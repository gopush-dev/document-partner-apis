1. Query string 연동 방식

Query string 연동방식

가장 단순한 방식입니다.

아래와 같이 대칭키 암호화가 적용된 user_id 를 전달해주시면 됩니다.

(암호화하지 않는 방식도 가능하지만 보안에 문제가 생길 수 있음) 

gopush://?method=login&user_id=2323929d02101d91023912039d19230

연동 단계

0. 연동 정보 요청

  • key: AES256 알고리즘에 사용할 대칭키
  • salt: 암호화 이전 user_id 앞에 붙일 prefix 정보

1. 로그인한 유저 ID 암호화(최신)

로그인한 유저 ID 앞에 Prefix 를 붙이고, AES256 알고리즘을 이용해 암호화합니다. 

// ex) AES256('testapp' + 'tester', key)
const user_id = AES256('Prefix' + user_id, key)

2. 로그인 메서드 호출

location 을 바꾸는 방법은 여러가지가 있겠지만 아래 예시에서는 location.href를 사용합니다.

앞서 암호화한 user_id 를 앱에 전달합니다.

location.href =  gopush://?method=login&user_id=2323929d02101d91023912039d19230

3. 연동 완료

주의할 점

1. 암호화 키, Salt 를 숨겨주세요

암호화 키나 Salt가 유출된다면 보안에 문제가 생기실 수 있습니다.
